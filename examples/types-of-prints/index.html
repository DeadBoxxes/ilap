what is the "PR" action?

The "PR" Action, short for "Print", is of of the main and best ways to display text on screen, there are many different kinds of PRs, here are a list of then

PR

basic printing text on screen

this method of print is the most simple with no extra elements.

PR"hello world"
SV"helloworld.ilap"
RUN"helloworld.ilap

OUTPUT:

hello world

PRLNBR/LNBR (Line break)

Adds a break in between the text

A much more sophisticated way of "decorating" prints, there are 3 examples i'll show, example one using PRLNBR, two being the usage of just LNBR to break the lines, and three being how to add a 1 line gap between text.
  • Example 1

    • this example only adds a break between text, similar to how the "br" element works in HTML

    PRLNBR"hello world"
    PR"down here!"
    SV"helloworld.ilap"
    RUN"helloworld.ilap

    OUTPUT:

    hello world
    down here!

  • Example 2

    • This example shows how to add a break without using PRLNBR, instead, using a regular break (LNBR), not specified as a print, this makes things more complicated, and i dont recommend it...

    PR"hello world"
    LNBR
    PR"down here, still!"
    SV"helloworld.ilap"
    RUN"helloworld.ilap

    OUTPUT:

    hello world
    down here, still!

  • Example 3

    • However, using LNBR by itself to print text CAN be helpful, take this for example, its used to add a gap between PRs, NOTE: remember to make the text printed BEFORE the LNBR in this tutorial a PRLNBR

    PRLNBR"hello"
    LNBR
    PR"world"
    SV"helloworld.ilap"
    RUN"helloworld.ilap

    OUTPUT:

    hello

    world

    Flashing PR

    This is not a "type" of PR, but this is a way to make your text "flashy", no pun intended.

    this tutorial requires a bit of code, so if you arnet comfortable with labels and other more "advanced" things, i suggest you know a bit more about ilap before trying to mess around with this straight away...

    NOTE: the way this works in html might not be supported by every browser, unfortunatley the blink element has been widely retired, but its the best way i can show this without loosing even more site compatibility...

    PR"flash" SETNAME"bl pr"
    NAMEDEF"bl pr" START:
    ACTWHEN"RUN"
    STARTLOOP:
    TIME"1000"
    DISPLAYNAME 1 "bl pr"
    TIME"1000"
    DISPLAYNAME 0 "bl pr"
    :STOPLOOP
    :NAMEDEF"bl pr" END
    SV"blinkingtest.ilap"
    RUN"blinkingtest.ilap"

    OUTPUT:

    flash

    LPR

    using the LPR (loop print) feature

    this tutorial shows you how to loop PRs win very little time, unlike the last tutorial, this one is super simple, and a very easy way to clogg up someones computer memory!
    the "=9" after the "LPR"texthere"" can have its value replaced:

    LPR"some list things"=9
    LNBR
    SV"loop.ilap"
    RUN"loop.ilap

    OUTPUT:

    some list things
    some list things
    some list things
    some list things
    some list things
    some list things
    some list things
    some list things
    some list things


    BUT! say you wanted to make a numbered list, little did you know, i thought that through! ill provide 2 examples showing these capibilites...

  • Example 1

    • CHNG=X (+)1 ACTWHEN"LPR" LPR"[X]. thing"=5
    LNBR
    SV"loop.ilap"
    RUN"loop.ilap

    by setting the actwhen for CHNG=X as "LPR", the system already knows to change the X value by 1 per every loop... this is a little trick i call "planning ahead"... NOTE: X can be replaced with any letter, just remember to put it in squared brackets ([]) when X is in a PR action.

    OUTPUT:

    1. thing
    2. thing
    3. thing
    4. thing
    5. thing

  • Example 2

    • you can also use this for dates, no, no... not relationships, then she'll think you are a nerd!!! but instead for writting days of the year, this example will even show how to start at a specific day/value before the loop... in this example the day will start at 7. NOTE: you still have to specify an ACTWHEN when pre setting the value, i suggest setting the ACTWHEN to preform the action on run, unless you have other plans...

    SET=X 7 ACTWHEN"RUN" CHNG=X (+)1 ACTWHEN"LPR" LPR"April [X]"=12
    LNBR SV"loop.ilap"
    RUN"loop.ilap

    OUTPUT:

    April 7
    April 8
    April 9
    April 10
    April 11
    April 12
    April 13
    April 14
    April 15
    April 16
    April 17
    April 18
