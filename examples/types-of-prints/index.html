what is the "PR" action?

The "PR" Action, short for "Print", is of of the main and best ways to display text on screen, there are many different kinds of PRs, here are a list of then

PR

basic printing text on screen

this method of print is the most simple with no extra elements.

PR"hello world"
SV"helloworld.ilap"
RUN"helloworld.ilap

hello world

OUTPUT:

hello world

PRLNBR (Line break)

Adds a break in between the text

A much more sophisticated way of "decorating" prints, there are 3 examples i'll show, example one using PRLNBR, two other just using LNBR to break the lines, and three being how to add a 1 line gap between text.
  • Example 1

  • this example only adds a break between text, similar to how the "br" element works in HTML

    • PRLNBR"hello world"
    PR"down here!" SV"helloworld.ilap"
    RUN"helloworld.ilap

    OUTPUT:

    hello world
    down here!

  • Example 2

    • This example shows how to add a break without using PRLNBR, instead, using a regular break (LNBR), not specified as a print, this makes things more complicated, and i dont recommend it...

    PR"hello world"
    LNBR
    PR"down here, still!"
    SV"helloworld.ilap"
    RUN"helloworld.ilap

    OUTPUT:

    hello world
    down here, still!

  • Example 3

    • However, using LNBR by itself to print text CAN be helpful, take this for example, its used to add a gap between PRs, NOTE: remember to make the text printed BEFORE the LNBR in this tutorial a PRLNBR

    PRLNBR"hello"
    LNBR
    PR"world"
    SV"helloworld.ilap"
    RUN"helloworld.ilap

    OUTPUT:

    hello

    world

    Flashing PR

    This is not a "type" of PR, but this is a way to make your text "flashy", no pun intended.

    this tutorial requires a bit of code, so if you arnet comfortable with labels and other more "advanced" things, i suggest you know a bit more about ilap before trying to mess around with this straight away...

    NOTE: the way this works in html might not be supported by every browser, unfortunatley the blink element has been widely retired, but its the best way i can show this without loosing even more site compatibility...

    PR"flash" SETNAME"bl pr" NAMEDEF"bl pr" START:
    ACTWHEN"RUN"
    STARTLOOP:
    TIME"1000"
    DISPLAYNAME 1 "bl pr"
    TIME"1000"
    DISPLAYNAME 0 "bl pr"
    :STOPLOOP
    :NAMEDEF END
    SV"blinkingtest.ilap"
    RUN"blinkingtest.ilap

    OUTPUT:

    flash
